1 · Overview

The Design of Experiments app generates structured sets of experimental conditions for materials science research. Instead of testing variables one at a time, DoE lets you explore the full parameter space efficiently — covering more combinations with fewer experiments.

The app is organised into three tabs:

  • Setup & Variables — choose algorithm, sample count, and define your parameters
  • Results & Visualization — review the generated design and explore it interactively
  • Export — download the table for instruments or a readable protocol for the lab
Reproducibility: Every generated design is tied to a random seed. Record the seed alongside your results so the exact design can be regenerated later.

2 · Tab 1 — Setup & Variables

Tab 1
Setup & Variables

2.1 Algorithm

Select a sampling algorithm from the dropdown. A short description appears immediately below the selector. See Section 5 for a comparison of all algorithms.

2.2 Sample Configuration

Sample Size

Use the slider to set the number of experimental conditions to generate. Range: 4 to 500. The slider minimum automatically adjusts to the recommended minimum for the number of variables you have defined (n_variables², minimum 4).

Random Seed

An integer seed that controls the random number generator. The same seed + same settings always produces the same design. Click New Seed to pick a random value, or type one directly. The seed range is 1 – 2 147 483 647 (full 31-bit range).

Advanced Options

A collapsible accordion containing algorithm-specific settings:

  • Latin Hypercube Sampling — optional centroid optimisation (None, random-cd, lloyd)
  • Sobol / Halton — scrambling checkbox (recommended; improves uniformity)
  • Other algorithms — no extra options

2.3 Variable Configuration

Define the experimental parameters (factors) you want to vary. Two empty rows are pre-filled. Use Add Variable to add more; Clear All to start over.

Each row has:

  • Variable Name — a unique identifier (letters, numbers, underscore, hyphen; must start with a letter; max 50 characters)
  • Type — Discrete, Continuous, or Categorical (see Section 6)
  • Parameters — Min / Max / Step for numeric types; comma-separated values for categorical
  • Description (optional) — appears in the exported protocol
Validation: A row highlighted with a red border means Min ≥ Max. Fix the range before generating — invalid variables are silently skipped.
Comma decimal separator: Both 1.5 and 1,5 are accepted in the Min / Max / Step fields.

2.4 Generate

Click Generate Samples to run the selected algorithm. The button is disabled during generation and re-enabled when done. A progress indicator appears while generating. On success the app automatically switches to Tab 2.

2.5 Quality Metrics (Tab 1, bottom)

After generation a scrollable panel shows space-filling quality scores for the design (see Section 7).

3 · Tab 2 — Results & Visualization

Tab 2
Results & Visualization (badge shows sample count after generation)

3.1 Info Bar

The top bar shows the seed, sample count, and algorithm used for the current design. The New Design button randomises the seed and immediately regenerates — useful for quickly exploring alternative designs under the same settings.

3.2 Generated Samples Table

Displays up to 50 rows of the generated design. The index starts at 1. Numeric columns with only integer values show no decimal places (e.g. 3); columns with fractional values show two decimal places (e.g. 1.25). Categorical columns show the category labels.

The full dataset (all rows) is available for download on Tab 3.

3.3 Visualizations

Select a plot type from the dropdown. Available types depend on the number of numeric variables:

Plot Type Requires Description
Scatter Plot Matrix ≥ 2 numeric vars All pairwise scatter plots. Best for spotting correlations and fill patterns.
Parallel Coordinates Any variables Each variable is a vertical axis; each sample is a polyline. Good for high-dimensional designs.
Distribution Plots Any variables Histogram (numeric) or bar chart (categorical) per variable. Shows marginal distributions.
3D Scatter Plot ≥ 3 numeric vars Interactive 3-axis scatter using the first three numeric variables.

Space-Filling Quality Any variables Four-panel diagnostic: distance distribution, coverage, uniformity, 2-D scatter.

Click Export Plot to download the current figure as a self-contained HTML file (includes Plotly; no internet needed to view).

3.4 Summary Statistics

A scrollable table at the bottom of Tab 2 shows per-variable statistics: type, minimum, maximum, and number of unique values.

4 · Tab 3 — Export

Tab 3
Export

Two distinct export modes are available:

4.1 Export Table

The sample matrix in machine-readable form, suitable for analysis software and lab automation instruments. Choose a format from the dropdown, then click Export Table:

  • CSV — plain text, one row per sample, comma-separated; opens in Excel, pandas, Origin, etc.
  • JSON — includes the variable definitions, algorithm name, seed, and timestamp alongside the samples; suitable for re-importing the design.
  • Excel — three sheets: Samples, Variables, Metrics. Requires openpyxl.

4.2 Readable Protocol

A numbered step-by-step document intended for a lab technician to follow manually. Each experiment is listed with its exact parameter values. Check Randomize experimental order to shuffle the execution sequence (useful to prevent systematic time-of-day or instrument-warm-up biases). Click Export Protocol to download as a plain-text .txt file.

Export Table vs Readable Protocol: Export Table is for computers and instruments; Readable Protocol is for humans in the lab.

5 · Sampling Algorithms

Algorithm Best for Notes
Latin Hypercube Sampling General screening; response surface methodology Guarantees each parameter is sampled uniformly across its range. Good default choice.
Sobol Sequences Sensitivity analysis; computational experiments Low-discrepancy quasi-random sequence. Excellent space-filling; prefer power-of-2 sample sizes.
Halton Sequences Optimization studies requiring consistent coverage Quasi-random; avoids clustering. Similar to Sobol but based on prime-number bases.
Random Sampling Baseline comparison; unbiased exploration No structure imposed. Clustering is possible, especially with small sample sizes.
Uniform Grid Sampling Systematic mapping of a complete response surface All combinations at regular intervals. Sample count must be a perfect n-th power of the number of variables; the app rounds up automatically.
Maximin Distance Design Expensive experiments where each test must provide maximum information Iteratively optimises the minimum inter-sample distance. Computationally heavier than other methods; slower for large n.
Rule of thumb: Use Latin Hypercube Sampling unless you have a specific reason to choose otherwise. Uniform Grid grows exponentially with the number of variables — avoid it with more than 3–4 variables.

6 · Variable Types

Continuous

Any real value between Min and Max. Use for temperature, pressure, flow rate, concentration, etc. Parameters: Min, Max.

Example: Temperature, 100, 300 → samples drawn from [100, 300].

Discrete

Integer or evenly-spaced values defined by Min, Max, and Step. Use for number of layers, spin speed in fixed increments, annealing cycles, etc. Parameters: Min, Max, Step (default 1).

Example: Layers, 1, 5, 1 → valid values {1, 2, 3, 4, 5}.

The table shows discrete values as whole numbers (3, not 3.00) when all values in the column are integers.

Categorical

Named categories with no numeric ordering. Use for material type, solvent, substrate, catalyst, etc. Parameters: a comma-separated list of at least two unique labels.

Example: Solvent, DMF, DMSO, GBL → each sample is assigned one of the three labels.

7 · Quality Metrics

After generation, the following metrics assess the space-filling quality of the design:

Metric Higher is better? Meaning
Minimum Distance Yes Smallest Euclidean distance between any two samples in the normalised unit hypercube. Higher means samples are more spread out.
Mean Distance Yes Average pairwise distance. Provides a global view of separation.
Distance Uniformity Lower is better Coefficient of variation of pairwise distances. Lower means more even spacing.
Overall Coverage Yes Fraction of the design space utilised, averaged across variables.
Max Correlation Lower is better Largest absolute Pearson correlation between any two numeric variables. Near 0 is ideal.

The Space-Filling Quality plot on Tab 2 shows these metrics graphically: distance histogram, per-variable coverage bars, uniformity scores, and a 2-D scatter of the first two numeric variables.

8 · Typical Workflow

  1. Define variables. Go to Tab 1. Add the parameters you want to study, choose their types, and set meaningful ranges. Watch for red-bordered rows (invalid Min/Max) and fix them.
  2. Choose algorithm and sample size. Latin Hypercube Sampling with 10–20× the number of variables is a good starting point. Adjust the seed if you want a specific reproducible run.
  3. Generate. Click Generate Samples. The app switches automatically to Tab 2.
  4. Review the design. Check the sample table, inspect the Scatter Plot Matrix for clustering, and look at the quality metrics. If the design looks poor, click New Design to get a different random realisation without leaving Tab 2.
  5. Export. Go to Tab 3. Download the CSV/Excel for analysis software, and/or the Readable Protocol for your lab notebook.

9 · Tips & Best Practices

Sample size

  • Minimum rule: at least n² samples for n variables.
  • For a response surface model: 10–20× the number of variables.
  • For a quick screening: 5–10× the number of variables.
  • Sobol sequences work best at power-of-2 sizes (8, 16, 32, 64 …).

Algorithm choice

  • Start with Latin Hypercube Sampling — balanced and fast.
  • Use Sobol / Halton for sensitivity analysis or when you need near-perfect uniformity.
  • Avoid Uniform Grid with more than 3 variables unless you want a very small number of levels per variable.
  • Maximin Distance is best for expensive experiments (slow synthesis, long characterisation) where every data point must count.

Reproducibility

  • Record the seed shown in the Tab 2 info bar alongside your experimental data. The JSON export includes the seed automatically.
  • The New Design button automatically picks a new random seed; the seed is always visible so you can reproduce any design.

Categorical variables

  • Most algorithms treat categories as integers internally and then map back to labels — the assignment is fair but not combinatorially exhaustive. Use Uniform Grid if you want every category to appear the same number of times.
  • Correlation metrics and the 3-D Scatter / Correlation Heatmap plots only use numeric variables; categorical variables appear in Distribution Plots and Parallel Coordinates.

Validation (red borders)

  • A red border on a variable row means Min ≥ Max. The variable is skipped during generation without an error — check the table to confirm all expected columns are present.
